ModusToolbox USBFXStack

The USBFXStack Middleware Library provides a powerful and versatile solution designed to support the EZ-USB(TM) FX2G3 device with a variety of integrated modules. This library includes essential drivers, utilities, and an operating system to help developers efficiently manage and optimize their applications.
As best practices and development flows are continuously evolving, it is recommended to periodically sync the latest updates from this library to ensure you are utilizing the most recent improvements and features.

FX devices

The EZ-USB(TM) FX2G3 family of USB2 device controllers is designed to meet the demands of established USB2 applications in biometrics, scanners, cameras, video, and imaging markets. FX2G3 leverages Cortex(R)-M4 (CM4) and Cortex(R) M0+ (CM0) microcontrollers, providing robust features such as 512 KB flash memory, 128 KB SRAM, and 128 KB ROM.
The controllers also include Serial Communication Blocks (SCBs), a crypto engine for enhanced security, and a high-bandwidth data subsystem that facilitates DMA data transfers from LVCMOS input to USB output at speeds up to 480 Mbps, suitable for USB Hi-Speed based host systems. A 1024 KB SRAM is integrated within the high-bandwidth data subsystem to provide ample buffering for data.

Features

  • FreeRTOS operating system: Includes source files for FreeRTOS, offering a reliable and widely-used real-time operating system.
  • LVDS/LVCMOS IP driver: Provides a driver for the Sensor Interface Port (SIP) on the FX2G3, configurable to operate using the LVCMOS interface for efficient data transfer.
  • USB stack: An integrated USB stack that supports Hi-Speed functionality on the FX2G3 device, enabling robust USB communication capabilities.
  • DMA manager: Offers a generic DMA channel abstraction for managing DMA resources, facilitating data movement across USB and LVCMOS interfaces on the FX2G3.
  • Utility functions: A set of utility functions providing core functionalities such as logging support, USB data transfer abstraction, and fault handling mechanisms.
